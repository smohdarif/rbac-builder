## 1. PackageGenerator Service

**File:** `services/package_generator.py`

### Class: `PackageGenerator`

```python
class PackageGenerator:
    """
    Generates a self-contained client delivery ZIP package.

    The ZIP contains individually-formatted API-ready JSON files,
    a Python deployment script, settings template, and README.
    """

    def __init__(self, payload: DeployPayload, project_key: str):
        self.payload = payload
        self.project_key = project_key
        self.customer_slug = payload.customer_name.lower().replace(" ", "_")

    def generate_package(self) -> bytes:
        """
        Build the complete ZIP package and return as bytes.
        Ready to pass directly to st.download_button(data=...).
        """

    def _build_role_files(self) -> List[Tuple[str, str]]:
        """
        Returns list of (filename, json_content) for each role.
        Files are numbered: 01_create-flags.json, 02_update-flags.json, etc.
        Content is stripped to only LD API-required fields.
        """

    def _build_team_files(self) -> List[Tuple[str, str]]:
        """
        Returns list of (filename, json_content) for each team.
        Files are numbered: 01_voya-web-dev.json, etc.
        """

    def _build_deploy_script(self) -> str:
        """Returns the complete deploy.py script as a string."""

    def _build_settings_template(self) -> str:
        """Returns settings.json template with placeholder API key."""

    def _build_readme(self) -> str:
        """Returns README.md using doc_generator."""

    def _build_rollback_template(self) -> str:
        """Returns empty rollback.json template (populated by deploy.py at runtime)."""
```

### Role file builder — fields included/excluded

| Field | Include in role file? | Reason |
|-------|-----------------------|--------|
| `key` | ✅ | Required by LD API |
| `name` | ✅ | Required by LD API |
| `description` | ✅ | Optional but useful |
| `base_permissions` | ✅ | Required — must be `"no_access"` |
| `policy` | ✅ | Required by LD API |
| `metadata` | ❌ | rbac-builder internal field only |

### Team file builder — fields included/excluded

| Field | Include in team file? | Reason |
|-------|-----------------------|--------|
| `key` | ✅ | Required by LD API |
| `name` | ✅ | Required by LD API |
| `description` | ✅ | Optional but useful |
| `customRoleKeys` | ✅ | Required by LD API |
| `roleAttributes` | ✅ | Required for attribute-based scoping |

## 2. deploy.py Script Design

The generated `deploy.py` is a **standalone Python script** with no dependencies except `requests`.

### Script structure

```python
#!/usr/bin/env python3
"""
LaunchDarkly RBAC Deployment Script
Generated by RBAC Builder — {date}
Customer: {customer_name}

Usage:
    pip install requests
    python deploy.py

Options:
    Set "dry_run": true in settings.json to preview without making API calls.
"""

import json
import os
import sys
import time
from pathlib import Path


# ─── Config ───────────────────────────────────────────────────────────────────

BASE_DIR = Path(__file__).parent
SETTINGS_FILE = BASE_DIR / "settings.json"
ROLES_DIR = BASE_DIR / "01_roles"
TEAMS_DIR = BASE_DIR / "02_teams"
ROLLBACK_FILE = BASE_DIR / "rollback.json"


# ─── LD API client (inline, no imports) ───────────────────────────────────────

class LDClient:
    def __init__(self, api_key: str, base_url: str):
        ...

    def create_role(self, role: dict) -> dict:
        """POST /api/v2/roles"""

    def create_team(self, team: dict) -> dict:
        """POST /api/v2/teams"""

    def role_exists(self, key: str) -> bool:
        """GET /api/v2/roles/{key} → 200 means exists"""

    def team_exists(self, key: str) -> bool:
        """GET /api/v2/teams/{key} → 200 means exists"""


# ─── Deployment logic ─────────────────────────────────────────────────────────

class Deployer:
    def __init__(self, settings: dict):
        ...

    def run(self):
        """Main entry point — deploys all roles then all teams."""

    def _deploy_roles(self) -> DeployResult:
        """Load and create all files from 01_roles/ in filename order."""

    def _deploy_teams(self) -> DeployResult:
        """Load and create all files from 02_teams/ in filename order."""

    def _load_json_files(self, directory: Path) -> list[tuple[str, dict]]:
        """Load all .json files sorted by filename."""

    def _write_rollback(self, created_roles: list, created_teams: list):
        """Write rollback.json with all successfully created resource keys."""


# ─── Result tracking ──────────────────────────────────────────────────────────

class DeployResult:
    created: list[str]     # keys successfully created
    skipped: list[str]     # keys that already existed (409)
    failed: list[str]      # keys that failed with error
    errors: dict[str, str] # key → error message


# ─── Entry point ──────────────────────────────────────────────────────────────

if __name__ == "__main__":
    settings = load_settings()
    deployer = Deployer(settings)
    deployer.run()
```

### deploy.py — API retry logic

| HTTP Status | Action |
|-------------|--------|
| 201 Created | ✅ Success — log created |
| 409 Conflict | ⚠️ Already exists — log skipped, continue |
| 429 Rate limit | ⏳ Wait 1 second, retry once |
| 401 Unauthorized | ❌ Abort immediately — bad API key |
| 403 Forbidden | ❌ Abort — insufficient permissions |
| 4xx other | ❌ Log error, continue to next resource |
| 5xx | ❌ Log error, continue to next resource |

## 3. settings.json Schema

```json
{
  "api_key": "YOUR_API_KEY_HERE",
  "base_url": "https://app.launchdarkly.com",
  "dry_run": false,
  "rate_limit_pause_seconds": 0.2
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `api_key` | string | required | LD API access token with Admin scope |
| `base_url` | string | `https://app.launchdarkly.com` | Override for custom/EU instances |
| `dry_run` | bool | `false` | Preview mode — no API calls made |
| `rate_limit_pause_seconds` | float | `0.2` | Pause between API calls to avoid rate limiting |

## 4. Individual Resource File Formats

### Role file (`01_roles/NN_<key>.json`)

```json
{
  "key": "update-targeting",
  "name": "Update Targeting",
  "description": "Template role for Update Targeting",
  "base_permissions": "no_access",
  "policy": [
    {
      "effect": "allow",
      "actions": ["updateOn", "updateRules", "updateTargets", "..."],
      "resources": [
        "proj/${roleAttribute/projects}:env/${roleAttribute/update-targeting-environments}:flag/*"
      ]
    },
    {
      "effect": "allow",
      "actions": ["viewProject"],
      "resources": ["proj/${roleAttribute/projects}"]
    }
  ]
}
```

### Team file (`02_teams/NN_<key>.json`)

```json
{
  "key": "voya-web-dev",
  "name": "Voya Web: Developer",
  "description": "Development team",
  "customRoleKeys": [
    "create-flags",
    "update-flags",
    "update-targeting",
    "apply-changes",
    "review-changes"
  ],
  "roleAttributes": [
    { "key": "projects", "values": ["voya-web"] },
    { "key": "update-targeting-environments", "values": ["production", "staging"] },
    { "key": "apply-changes-environments", "values": ["production"] }
  ]
}
```

## 7. Error Handling

### In PackageGenerator
- Empty payload → raise `ValueError("No roles or teams to package")`
- ZipFile write failure → propagate as `PackageGenerationError`

### In deploy.py (client-side)
- Missing `settings.json` → print clear error with instructions
- `api_key` still placeholder → print error: "Please set your API key in settings.json"
- Network error → retry once, then log and continue
- All roles failed → abort before creating teams (teams would fail anyway)

## 8. rollback.json Schema

Written by `deploy.py` after a successful deployment run:

```json
{
  "deployed_at": "2026-03-18T19:30:00",
  "customer": "Voya",
  "project": "voya-web",
  "created_roles": [
    "create-flags",
    "update-flags",
    "update-targeting"
  ],
  "created_teams": [
    "voya-web-dev"
  ],
  "rollback_commands": [
    "DELETE /api/v2/teams/voya-web-dev",
    "DELETE /api/v2/roles/create-flags",
    "DELETE /api/v2/roles/update-flags",
    "DELETE /api/v2/roles/update-targeting"
  ]
}
```

> Teams are listed before roles in `rollback_commands` — teams must be deleted first as they reference the roles.
