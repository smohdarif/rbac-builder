## UI Design: Default vs Optional Permissions

**Source:** `[Template] [S2] LD Custom Roles Planning.xlsx` — Settings tab, `Default` column.

The S2 template already defines which permissions to show by default vs hide:

### Default = 1 → Always visible in main matrix

| Project | Environment |
|---------|-------------|
| Create Flags ✅ | Update Targeting ✅ |
| Update Flags ✅ | Review Changes ✅ |
| Archive Flags ✅ | Apply Changes ✅ |
| Update Client Side Availability ✅ | Manage Segments ✅ |
| Manage Metrics ✅ | Manage Experiments ✅ |
| Manage Release Pipelines ✅ | View SDK Key ✅ |
| Manage Context Kinds 🔲 | |
| View Project ✅ | |

### Default = 0 → Hidden in `st.expander("Optional Permissions")`

| Permission | S2 Note | Type |
|------------|---------|------|
| Bypass Required Approvals | — | env |
| Manage Destinations | "Only used with data export" | env |
| Manage Triggers | "Guarded rollouts have largely replaced triggers" | env |
| Manage Holdouts | — | env |
| Export Segments | — | env |
| Update Flag Variations | "Included in update flags since this supports approvals" | project |
| Create+Update Flags (combined) | "For merging create/update into one role" | project |
